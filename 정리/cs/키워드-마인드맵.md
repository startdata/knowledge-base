# 기술 키워드 마인드맵 (통합본)

> 기술 스택에서 파생되는 모든 키워드를 **하나의 마인드맵**으로 통합한 문서다.
> 가지마다 깊이가 다르다 — 메커니즘이 깊은 주제는 **10단계 꼬리**까지, 얕은 주제는 4~5단계에서 자연 종료한다.
> (억지로 깊이를 늘리지 않았다. 멈춘 곳이 그 개념의 실제 심화 한계 깊이다.)

## 사용법 — 마인드맵으로 보기
- **Markmap**: 이 파일 전체를 [markmap.js.org](https://markmap.js.org)에 붙여넣으면 즉시 인터랙티브 마인드맵이 뜬다. (노드 접기/펼치기 가능)
- **VS Code**: "Markmap" 확장 설치 후 이 파일에서 미리보기.
- **Obsidian**: Markmap 플러그인.

## 범례
- ⭐ = 내 프로젝트와 직결되는 노드 (자주 묻는 핵심 지점)
- 🪝 = 10단계까지 내려가는 "깊은 꼬리" 시작점 (이해 깊이가 갈리는 지점)

# 🪝 우선 학습 — 10단계 깊은 꼬리

이해 깊이가 갈리는 지점. 위 트리에 통합돼 있으며, 아래는 빠른 참조용 단일 경로다.

1. **Kafka EOS**: Kafka → 전달 보장 → exactly-once → 트랜잭셔널 프로듀서 → transactional.id → 프로듀서 epoch → 좀비 펜싱 → epoch 증가 → 이전 epoch 거부 → read-process-write 원자성
2. **비관적 락 장애 전파** ⭐: 비관적 락 → SELECT FOR UPDATE → X Lock → 락 대기 큐 → 커넥션 점유 → 풀 고갈 → 커넥션 대기 → 응답 지연 → 타임아웃 → 캐스케이딩 장애
3. **격리수준/MVCC**: 격리 수준 → REPEATABLE READ → MVCC → read view → undo 버전 체인 → 가시성 판단 → current read → Next-Key Lock → Gap Lock → phantom 차단
4. **DynamoDB 핫파티션** ⭐: 파티션 키 → 핫 파티션 → 처리량 불균형 → write sharding → suffix → 샤드 수 → fan-out → 병렬 Query → 결과 병합 → 읽기 복잡도 트레이드오프
5. **Outbox/CDC** ⭐: 이중 쓰기 → Outbox → 트랜잭션 내 INSERT → 릴레이 → CDC → binlog(Debezium) → offset 추적 → at-least-once → 멱등 컨슈머
6. **RAG 벡터검색**: RAG → 벡터 검색 → 임베딩 → 코사인 유사도 → ANN → HNSW → 계층적 그래프 → ef_search → recall↔latency
7. **이벤트 루프**: Node.js → 이벤트 루프 → libuv → 페이즈 순환 → poll → I/O 콜백 → CPU 바운드 점유 → 콜백 지연 → 워커 스레드 → 메시지 패싱
8. **Redis 캐시 스탬피드** ⭐: 캐싱 → cache stampede → 동시 미스 → DB 부하 폭증 → 분산락 갱신 제한 → 확률적 조기 갱신(PER) → TTL 지터 → 핫키 선갱신
